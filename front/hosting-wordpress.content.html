Por qué importa el stack

WordPress es exigente.
El hosting WordPress ecológico tiene que serlo también.

WordPress hace entre 50 y 400 consultas SQL en una sola carga de página, dependiendo de los plugins instalados. Ejecuta PHP en cada request y genera decenas de transacciones en base de datos. Un hosting genérico con Apache intenta procesar todo esto de forma secuencial.

Nuestro stack invierte esa lógica: LiteSpeed maneja la concurrencia mejor que Apache con el mismo hardware. Redis elimina el 90% de las consultas SQL almacenando los resultados en RAM. Cloudflare CDN sirve los assets desde el nodo más cercano al visitante. Y PHP 8.2 ejecuta WordPress entre un 20% y un 35% más rápido que PHP 7.4.

El TTFB (Time to First Byte) es el primer dato que llega al navegador de tu visitante y el punto de partida de todos los Core Web Vitals. Reducir el TTFB de 500ms a 50ms puede mejorar el LCP en más de un segundo (lo suficiente para pasar de "rojo" a "verde" en PageSpeed Insights).

Lo que hace diferente un hosting WordPress verde de verdad

No es solo montar WordPress en un servidor. Es ajustar cada capa del stack para el comportamiento específico de WordPress.

LSCache: caché nativa para WP

LiteSpeed Cache es el plugin de caché más rápido disponible para WordPress. Gestiona caché de página completa, ESI (Edge Side Includes) para contenido dinámico parcial, prefetch de páginas y minificación de CSS/JS (todo desde el mismo servidor, sin pasos intermedios).

Caché en servidor, no en PHP

Redis: cero consultas repetidas

WordPress sin caché de objetos consulta la base de datos para cada valor de wp_options, cada término de taxonomía, cada post relacionado. Redis intercepta esas consultas y devuelve el resultado desde RAM en microsegundos (hasta un 90% menos de carga en MySQL).

Respuesta en <1ms vs 10–60ms en DB

PHP 8.2 + OPcache

PHP 8.2 con JIT activado y OPcache de 256MB preconfigura el código PHP de WordPress en memoria compilada. No se re-parsea en cada petición. El resultado es un tiempo de ejecución PHP entre 20% y 35% más rápido que en PHP 7.4.

+20–35% velocidad PHP

WooCommerce: el hosting que lo entiende

WooCommerce tiene requisitos de caché muy específicos. Las páginas de carrito, checkout y cuenta de usuario nunca deben estar cacheadas (tienen contenido dinámico por usuario). El catálogo de productos, las páginas de categoría y el resto sí deben cachearse agresivamente. LSCache tiene reglas nativas para esto, preconfiguradas en nuestros entornos WooCommerce. También gestionamos las sesiones de usuario y los fragmentos de carrito mediante caché selectiva ESI.

Core Web Vitals

Core Web Vitals: las métricas que Google usa para posicionar tu WordPress

Desde 2021, los Core Web Vitals son un factor de ranking activo. El hosting determina directamente dos de los tres indicadores clave.

LCP
Largest Contentful Paint

Tiempo que tarda en cargarse el elemento principal de la página (imagen hero, bloque de texto H1). Determina cuándo el usuario percibe que la página "ya cargó". El hosting y el TTFB son los factores más determinantes.

Objetivo Replanta < 2.5s ✓
CLS
Cumulative Layout Shift

Mide la estabilidad visual durante la carga. Imágenes sin dimensiones especificadas, fuentes que cambian el texto o ads que empujan el contenido generan CLS alto. Depende del código del sitio, no solo del hosting.

Objetivo Replanta < 0.1 ✓
INP
Interaction to Next Paint

Reemplazó al FID en 2024. Mide cuánto tarda una página en responder visualmente a una interacción del usuario. JavaScript bloqueante y plugins que saturan el hilo principal son la causa más común de INP alto.

Objetivo Replanta < 200ms ✓

¿Por qué importa para el SEO?

Google confirmó que los Core Web Vitals son un factor de ranking en su algoritmo (Page Experience Signal). Un sitio en verde en los tres indicadores tiene ventaja de posicionamiento sobre uno en rojo, especialmente en queries competidas donde otros factores (autoridad, contenido) están igualados. Un buen hosting resuelve el LCP directamente con un TTFB bajo. El resto depende del código del sitio (que es exactamente lo que resolvemos en diseño web sostenible).

Preguntas frecuentes

Lo que nos preguntan sobre Hosting WordPress

Respuestas claras sobre rendimiento, planes y compatibilidad.

¿Por qué un hosting WordPress especializado es mejor que uno genérico?

Un hosting genérico está configurado para cualquier tipo de web. Un hosting WordPress especializado ajusta cada capa del stack al comportamiento de WordPress: PHP configurado para la carga de plugins, MySQL optimizado para las consultas de wp_options, Redis preconfigurado y LiteSpeed con reglas específicas para el patrón de URLs de WordPress. La diferencia en rendimiento es de 3 a 10 veces.

¿Qué es LiteSpeed y por qué importa para WordPress?

LiteSpeed es un servidor web que maneja hasta 9 veces más solicitudes concurrentes que Apache con el mismo hardware, y con menor consumo energético. Para WordPress, incluye LSCache (el plugin de caché más eficiente disponible, que gestiona caché de página completa, caché de objetos, prefetch y minificación desde el propio servidor, sin pasos intermedios).

¿Funciona bien para WooCommerce?

Sí, es nuestro caso de uso más habitual. WooCommerce tiene requisitos de hosting muy específicos: las páginas de carrito, checkout y cuenta nunca deben estar cacheadas. LSCache tiene reglas nativas para WooCommerce que excluyen automáticamente esas páginas del caché de página completa, mientras cachean el catálogo de productos de forma agresiva. También gestionamos correctamente las sesiones de usuario.

¿Qué versión de PHP usan?

PHP 8.2 por defecto, con soporte para 7.4, 8.0 y 8.1 si lo necesitas. PHP 8.2 es entre un 20% y un 35% más rápido que PHP 7.4 para WordPress. Puedes cambiar la versión de PHP desde el panel de control sin tiempo de inactividad (útil cuando tienes plugins que aún no son compatibles con 8.2).

¿Incluye entorno de staging para WordPress?

Sí, en todos los planes. Un staging es una copia del sitio en producción donde puedes probar actualizaciones de plugins, cambios de tema o desarrollos nuevos antes de aplicarlos al site real. Esencial para cualquier WordPress con tráfico relevante (especialmente para WooCommerce, donde un plugin roto puede costar ventas).

¿Cómo afecta el hosting a los Core Web Vitals de WordPress?

El hosting determina directamente el TTFB (Time to First Byte), que es el punto de partida de todos los Core Web Vitals. Con LiteSpeed + Redis activos el TTFB se reduce de forma muy significativa frente a hosting convencional. El CDN de Cloudflare reduce también la latencia para visitantes de cualquier región. En la práctica, cambiar de hosting puede mejorar el LCP en más de un segundo (aunque el resultado final depende también del código y optimización del propio WordPress).

¿Puedo gestionar el hosting yo mismo?

Sí. Todos los planes incluyen acceso a cPanel para gestión de archivos, bases de datos, correo y subdominios. También hay un instalador de WordPress con un clic, gestión de versiones PHP y configuración del certificado SSL desde el panel. Si prefieres que gestionemos nosotros el entorno técnico, contamos con planes de mantenimiento mensual que lo incluyen.

El Plan Sauce (€12/mes) soporta 1 sitio WordPress e incluye 5 buzones de email. El Plan Roble (€17/mes) amplía capacidad para equipos pequeños. El Plan Cedro (€32/mes) está pensado para WordPress y WooCommerce con más exigencia técnica. Todos los planes incluyen staging. Puedes ver la comparativa completa más arriba.
